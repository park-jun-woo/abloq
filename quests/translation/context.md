# 번역 퀘스트 — 컨텍스트 규약

번역 에이전트에게 주입되는 공통 규약. parkjunwoo.com의 12언어 게시 절차를 박제한
것이다 — 언어 목록 자체는 인스턴스 blog.yaml `languages` 선언이 단일 출처이고,
Seed가 기본 언어를 제외한 전 언어의 아이템을 만든다.

## 불변 목록 (번역 금지 — 게이트가 기계 검사)

- **코드블록**: ``` 펜스 안은 주석·문자열까지 그대로 — translation-parity ④가
  바이트 동일 multiset로 비교한다.
- **이미지 경로**(③), **외부 링크 URL**(⑤): 원문과 양방향 multiset 일치 — 누락도
  주입도 FAIL이다. 링크의 표기 텍스트는 번역해도 된다.
- **front matter `date`·`lastmod`**(⑦ fm-mirror): 원문 값 그대로. 위조하면 FAIL.
- **slug**: 원문과 같은 유효 slug(slug-consistency) — 전 언어가 같은 URL 경로를 가진다.

## 구조 보존 (게이트가 기계 검사)

- **헤딩**: 전체 헤딩의 레벨 시퀀스를 보존한다(①). 인식 섹션(sources 등)은
  `structure.headings` 맵의 대상 언어 텍스트를 그대로 쓰고(##), 자유 헤딩은 번역하되
  레벨과 순서를 유지한다.
- **문단 분할 보존**(②): 빈 줄 경계 블록 수 1:1. 문단 경계는 레이아웃 속성이고
  번역자가 출력을 통제하므로 CJK(ja 등)·RTL(ar 등)에도 강제된다 — 문장을 합치거나
  쪼개지 마라.
- **내부 글 링크**(⑥): 같은 언어 버전으로 치환 — `/{대상언어}/...` 프리픽스 필수.
  기본 언어는 루트 서빙이라 원문 링크에는 프리픽스가 없을 수 있다. 이미지·정적 자산
  경로(확장자 있는 경로)는 치환하지 않는다(③의 불변 영역).

## 언어 특이 처리

- **고유명사·제품명·코드 식별자**는 번역하지 않는다.
- **RTL(ar·he 등)**: 본문은 대상 언어 문자로 자연히 RTL 렌더된다 — 마크다운 구조
  문자(`#`, `-`, `[`, `(`)와 URL·코드는 LTR 그대로 둔다. 방향 제어 문자
  (U+202A~U+202E, U+2066~U+2069) 삽입 금지 — 구조 비교를 오염시킨다.
- **CJK(ja·zh·ko)**: 문단 분할 보존 규약이 그대로 적용된다. 구두점은 대상 언어
  관행(。、 등)을 따르되 링크·코드 구문은 ASCII 그대로.

## 금지 사항 (치즈 방어)

1. **원문 수정 금지** — 이 퀘스트의 제출물은 번역 파일 1편뿐이다. 원문을 고쳐
   패리티를 맞추는 우회는 금지다. 원문은 Prepare가 매 제출 시점에 다시 읽으므로
   원문 훼손은 비교 기준만 바꿀 뿐이고, repo 레벨 게이트와 리뷰에서 드러난다.
2. **lastmod 위조 금지** — 번역의 date·lastmod는 원문 이식이 규약이고 fm-mirror(⑦)가
   기계 검사한다. 재동기화 시점의 lastmod 갱신은 갱신 퀘스트(Phase018)의 몫이다.
3. **범위 밖 파일 변경 금지** — blog.yaml·다른 글·레이아웃을 건드리지 마라.
4. **게이트 우회 금지** — hugo 빌드는 인스턴스 전체를 대상으로 매 제출 실행된다.
   hugo가 PATH에 없으면 제출 자체가 중단된다(스킵 아님).
