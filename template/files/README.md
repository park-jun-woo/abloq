# abloq blog

This repository was scaffolded by `abloq init`. It is an agent-operable blog:
**blog.yaml is the single source of truth**, derived files are generated, and
every article change is judged by deterministic gates.

## Layout

```
blog.yaml                  # SSOT — edit this, then run `abloq generate`
CLAUDE.md                  # agent operations manual (generated by abloq)
content/{lang}/{section}/  # articles
layouts/ assets/ static/   # layout pack (yours to edit; gates don't care)
data/jsonld.json           # generated — consumed by layouts/partials/jsonld.html
hugo.toml                  # generated — do not edit
static/robots.txt          # generated — do not edit
static/llms.txt            # generated — do not edit
quests/queue/              # work queue dropped by the ops backend
deploy/terraform/          # optional IaC (S3 + CloudFront + WAF + logs + IndexNow)
public/                    # hugo build output (gitignored)
```

## Daily loop

```bash
abloq validate          # blog.yaml schema + rules
abloq generate          # re-render hugo.toml / robots.txt / llms.txt / jsonld.json
abloq check             # drift check: derived files match blog.yaml
hugo                    # build to public/ (no --minify before gating:
                        # the hreflang gate reads the built HTML)
abloq gate              # structure + evidence gates over content/ and public/
abloq postbuild md      # noise-free .md next to every built article (AI context)
```

## Layout pack policy

The bundled layouts/CSS/JS are a **default layout pack extracted from
parkjunwoo.com**. They are *replaceable but unsupported*: you can swap in any
Hugo layout, and the gates and generators will keep working (they only touch
blog.yaml, content/ and the generated files) — but issues inside a custom
layout are yours. If you replace the layout, keep two contracts:

1. consume `data/jsonld.json` for JSON-LD (don't hardcode identity),
2. emit a self-inclusive set of `hreflang` alternate links on article pages
   (the `hreflang-complete` gate reads the built HTML).

Personal identity never lives in the layout: author, title, languages and
sections come from blog.yaml (via the generated hugo.toml params); social
links and the home banner are partial hooks under `layouts/partials/hooks/`.
